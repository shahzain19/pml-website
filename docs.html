Introduction

PML (Project Memory Layer) is a local-first authority system for AI-assisted development. It prevents context drift by maintaining a persistent graph of project rules, decisions, and structure.

Why PML?

LLMs context windows are ephemeral. Even with large windows, they forget the *reasoning* behind decisions made 2 weeks ago. PML creates a permanent record that AI tools must consult before suggesting changes.

$ npm install -g pml-cli

$ pml init

✓ Memory graph initialized at .pml/

✓ Scanned 45 files

✓ Generated .cursor/rules.md

Core Philosophy

Local First

Data never leaves your machine. Your graph is your own.

Deny by Default

Structural changes must be explicitly approved.

Tool Agnostic

Maintains authoritative files for Cursor, Copilot, and others.
